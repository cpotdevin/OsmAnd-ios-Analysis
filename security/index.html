Wire App Analysis・Security

To evaluate the security features in the Wire app we will investigate the two most important data management processes in a messaging app: sending messages and storing message information on the phone.

Scenarios

  • Android
    • Sending
    • Storing
  • iOS
    • Dependencies structure

Android Security

If we investigate the process of sending a message to the backend API we arrive to the method postOtrMessage() that encrypts the message content with the OtrService class.

iOS Security

Wire has a dependency called Cryptobox and is in charge of doing all the security things related with the app. More precisely, this dependency does all the hashing and encrypting in the app.

Also, creates the “fingerprint” of the device. This is used to verify from where has been sent a message, to prove the veracity of the sender. Also, this dependency can see the other “fingerprints” from other devices; from other users or the user himself. This is one of the biggest security functionalities that Wire provides in terms of security.
